CONTRACT MANDOR / SUB

 

 

Definisi

Contract adalah suatu dokumen perjanjian tertulis yang berisi mengenai data-data kesepakatan pekerjaan yang harus dikerjakan antara Pihak Pertama sebagai pemberi tugas (Entitas TJS), dengan Pihak Kedua sebagai pelaksana pekerjaan (Mandor, Subkont atau lainnya).

 

Tampilan layar List Contract  (klik tombol view )

Fungsi dan definisi tombol :

Klik                                : untuk membuat / menambah kontrak baru.

Ketik di kolom Search  : untuk mencari file kontrak.

Total                     : jumlah nilai kontrak (terisi otomatis)

Excel                     : untuk export list kontrak ke excel.

1,2,3,4, dst          : no lembar halaman.

Show ... entries   : jumlah baris / input kontrak per halaman.

Live Chat           : fasilitas live chat, untuk komunikasi user di IPMS.

 

 

 

 

Tampilan layar Contract di IPMS :

Fungsi dan definisi tombol

    : Untuk memilih file kontrak excel di lokal.

    : Untuk import kontrak dari file excel lokal ke IPMS.

 

 

 

 

 

 

 

 

 

 

 


Cara membuat Contract di IPMS :

Berikut langkah-langkah membuat Kontrak di IPMS yaitu :

1.    Langkah awal : masuk project.

( klik  Select Project , kemudian ketik nama project di kolom search,  klik Select sesuai project yang dipilih ).

2.    Masuk ke menu Contract.

( pilih / klik Contract / PO/DO, klik Contract).

3.    Pengisian Kelengkapan Kontrak

Adapun data-data yang harus dimasukan didalam Contract adalah :

Nomor kontrak                  : diisi nomor kontrak

Tanggal kontrak                 : diisi tanggal kontrak

Kepada                      : dipilih penerima tugas (perusahaan/perorangan),

Type contract                     : dipilih salah satu

·      Consultant Contract    :  ......

·      Consultant Agreement :  .......

·      Mandor                :  untuk yang mengerjakan kontrak upah.

·      Other-Commited cost :  .......

·      Professional Services  :  .........

·      Purchase Order  : ......

·      Subcontractor     : untuk yang mengerjakan kontrak

§  upah dan bahan.

4.    Selanjutnya klik tombol save  : untuk menyimpan informasi kontrak.

 

Note : Apabila Nama Instansi dan Pemberi Tugas belum ada di daftar, mohon diinfo ke group telegram TJS Discusstion, agar diinput TIM IPMS

6.    Tab Detail Kontrak

Tab Detail Kontrak adalah form untuk pengisian detail kontrak pekerjaan seperti memasukkan uraian pekerjaan, volume, harga satuan dll.

Dalam tab ini juga digunakan untuk edit, menambah dan mengurangi item pekerjaan di kontrak, dan lain-lain.

 

Tampilan Tab Detail Kontrak

Langkah-langkah pengisian / input detail kontrak yaitu :

·      Langkah awal simpan dulu kontrak yang dibuat (klik save )

·      Buat uraian item kontrak di Excel sesuai template contract yang ada (download template contract di OneDrive)

 

Catatan :  untuk tambahan / edit uraian kontrak dan kelengkapan pengisian kolom, bisa dilanjutkan di IPMS dengan klik Add Row atau Delete Row.

Masukkan file kontrak tersebut (klik Import File)

Fungsi   untuk salin dan export file kontrak yang sudah dibuat sesuai type filenya.

 

Tab INCLUSION/EXCLUSION

Form untuk pengisian, item pekerjaan yang termasuk dan di luar kontrak

 

Tab  ini berguna untuk memasukan data-data kondisi yang berkaitan dengan kontrak yang sudah diterbitkan,

Inclusions (termasuk), adalah kondisi pekerjaan yang sudah masuk didalam scope kontrak yang diberikan, misalnya, alat kerja standar, PPh 21, Retensi, dll.

Exclusions (tidak termasuk), adalah kebalikan dari inclusions, misalnya, peralatan besar seperti molen, vibrator, alat safety, dll.

 

Tab PENGURANGAN DP/RETENSI

Fungsi tab pengurangan dp/retensi adalah form untuk pengisian prosentase DP, agar saat melakukan tagihan invoice bisa otomatis dipotong pengembalian DP nya.

Pengesahan

Lembar pengesahan berisi : Dibuat Oleh, Diverifikasi Oleh, Disetujui Oleh, Disahkan Oleh, terisi otomatis setelah didaftarkan sebagai user di IPMS.

Status dan tanggal TTD , dibuat berjenjang dari level terbawah sampai terakhir.

 

Selanjutnya setelah semua data terisi dan data di save, di pengesahan klik  (Belum TTD) warna merah, akan menjadi (Sudah TTD) warna hijau.
